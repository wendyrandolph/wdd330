Testing & Debugging

There are three types of errors in programming:

  • System Error - There's a problem with the system or external
    devices with which the program is interacting.
  • Programmer Error - The program contains incorrect syntax or faulty logic; could be as simple as a typo.
  • User Error - User has entered data incorrectly.

Exceptions

  • An error that produces a return value that can then be used by the program to deal with the error.
  • ex. unicorn();
  • << ReferenceError:unicorn is not defined.

Stack Traces

  • A sequence of functions or method calls that lead to the point where the error occured.

Warnings: will occur if there's an error in the code that isn't enough to cause a crash.

Importance of Testing and Debugging

  • Strict Mode: 'use strict'; (added to the first line of javascript.
  • Linting Tools: can test quality of javascript file. Designed to highlight sloppy programming practices.
  • Debugging in the Browser: Process of finding out where bugs occur.
  • Trusty Alert: alert()
  • Console:
    • console.log()method
    • console.trace()method - will log an interactive stack trace in the console.
  • Error Objects - ex. const error= new Error("Oops, Something went wrong")

Throwing exceptions will allow any problems to be highlighted.

ex. throw new Error('something went wrong');

Exception Handling:

When an exception occurs, the program terminates with an error message.

Tests: writing good tests means your code will be less brittle as it develops.

Test Driven Development (TDD)

  1. Write tests that initially fail.
  2. Write code to pass the tests.
  3. Refactor the code.
  4. Test refactored code.
  5. Write more tests for more features.

Testing Frameworks

Jest - TDD framework. Install using npm ( npm install -g jest).
